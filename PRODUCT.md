# Product

## Register

product

## Users

ML engineers, CUDA/Triton learners, and attention-paper readers use this as a focused reference while studying how equations map onto implementation.

## Product Purpose

Broq Cheatsheet presents attention algorithms as compact paper-style math beside syntax-highlighted code. Success means users can switch variants, inspect equations, and understand the corresponding implementation path without losing context.

## Brand Personality

Precise, calm, technical. The interface should feel like a trustworthy study bench: dense enough for experts, clear enough for learners, and restrained enough to keep the math and code primary.

## Anti-references

Avoid marketing-page composition, decorative dashboards, oversized cards, playful visual metaphors, and controls that look like loose text rather than working product navigation.

## Design Principles

- Keep equations and code as the visual priority.
- Make navigation and controls feel familiar, compact, and immediately operable.
- Preserve paper-and-terminal contrast without adding decorative noise.
- Prefer fixed, stable component geometry so switching examples does not shift the workspace.
- Use accent color for selection, focus, and state rather than decoration.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. Preserve keyboard focus states, tab semantics, reduced-motion behavior, and readable layouts on narrow screens.
